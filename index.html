HOW TO USE THIS SITE

Use this page as the fast setup guide before opening the dossier or referee console. It reflects the current live workflow: guest entry or Google login, linked rooms, GM character tabs, local NPC tabs, and dossier-side saves.

// 01 - START HERE

Choose Your Entry Point

Use Load File for an existing dossier, Create Character for a blank sheet, or Open GM Page for the referee console. If you are not signed in, the site now lets you choose Log In With Google or Enter Without Login before continuing.

// 02 - ROOM LINK

Connect Players & GM

Open the GM page, enter a room ID, then connect player dossiers using that same ID. Connected players appear as their own GM tabs instead of a simple monitor list.

// 03 - ACCOUNT SAVE

Login Or Enter Guest

Google login is for account-linked saves. Signed-in users can reopen saved dossiers from the homepage and save from inside the dossier. Guest entry is still valid if you only need a quick local session and do not need account restore.

// 04 - FILES

What You Can Import

The site supports character .txt, item .txt, special-skill .txt, and bundled dossier .zip files. Use the format guide on the load card as the template.

// 05 - SESSION FLOW

Typical Use

Load or create characters, connect the room, use the dossier for player-side tracking, and use the GM page for live player tabs, remote breach control, NPC handling, and session-side reference. Signed-in players can also use Load From Save inside the dossier drawer.

// 06 - NPC / MOOKS

Generate With AI

For fast NPCs, mooks, or one-off enemies, use an external chatbot AI, format the result to match the upload template, then import it as .txt or bundled .zip. GM local NPC tabs can now persist to the signed-in account too.

// 07 - BUILD NOTE

Development

Built with roughly 80% AI assistance. For practical use, the important part is the workflow: upload correctly formatted files, link the room when needed, use external chatbot generation for random content, and sign in only when you want account-backed saves.

After you upload or confirm a new character, the system runs a short "CHIPPIN IN" glitch load before opening the dossier.

CHARACTER / ITEM FORMAT

  "Character Name", "Street Name"
}

  REF=0, INT=0, COOL=0, ATTR=0, TECH=0, LUCK=0, MA=0, BODY=0, EMP=0

  "Solo"
}

  point=0
